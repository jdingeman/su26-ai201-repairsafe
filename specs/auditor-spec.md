## Purpose

Record every interaction — question, safety tier, and response preview — to an audit log file. Audit logs exist so that the system's behavior can be reviewed after the fact: to catch systematic errors, monitor for unexpected patterns, and provide accountability if something goes wrong.

---

## Input / Output Contract

**Inputs:**

| Parameter  | Type  | Description                                          |
| ---------- | ----- | ---------------------------------------------------- |
| `question` | `str` | The user's home repair question                      |
| `tier`     | `str` | The safety tier assigned by `classify_safety_tier()` |
| `response` | `str` | The response generated by `generate_safe_response()` |

**Output:** `None` — side effects only (writes to file, prints to terminal)

### Why these truncation limits?

_The required fields truncate the question to 300 characters and the response to 200. Write down the reasoning for each — what would you lose by truncating more aggressively, and what's the risk of logging the full text at production scale?_

```
The truncation prevents enormous data buildup for a log file, such as 8MB of response text daily. It becomes more challenging to load the detail into a spreadsheet for manual review. It also reduces surface area for private information to be logged, in case a user decided to add their address or PII into the question.
The risk for the question truncation is that the text to be cut off is likely what determined the safety tier. The risk for the response truncation is that the text may have leaked instructions later on after telling the user to contact a professional. The response_length field can be used to investigate refuse-tier responses with unusually long responses.
```

### Directory creation

_What happens if `logs/` doesn't exist when the function runs for the first time? How will you handle that — and why is this worth thinking about at all?_

```
The logs directory should be created in the case that it doesn't already exist. Otherwise, it should just be written to. This is worth thinking about so that the program does not crash in case the logs get deleted at any point.
```
